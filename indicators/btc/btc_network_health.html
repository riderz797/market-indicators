Bitcoin Network Health Index BTC

Composite 0–100 score across 11 on-chain metrics in five dimensions: security budget, decentralization, miner sustainability, real usage, and holder structure. Each metric is percentile-ranked against its own 4-year rolling history — measuring health relative to baseline, not absolute size.

METHODOLOGY

Construction

1 Fetch daily data for 11 metrics from free public APIs
2 Apply 30-day moving average to each raw series (noise removal)
3 Percentile-rank each metric against its own trailing 4-year (1,460-day) window → 0–100 sub-score
4 Invert pool concentration and LTH proxy so 100 always = healthier
5 BNHI = weighted sum of sub-scores (weights visible above, adjustable)

Data Sources

blockchain.info — miners-revenue, transaction-fees-usd, hash-rate, n-unique-addresses, estimated-transaction-volume-usd, market-cap, coin-days-destroyed

bitnodes.io — reachable full-node count (daily snapshots)

mempool.space — per-pool hashrate share history (top-2 pool concentration)

All sources are free with no API keys. Run python build_bnhi.py to refresh.

Required Caveats

  • 1 Percentile-ranking is self-referential: during a sustained regime shift (e.g. post-ETF hashrate step-change), all metrics converge toward 50 as the window fills with the new baseline. The index measures relative health vs. recent history, not absolute health.
  • 2 Node count and pool concentration are weakly measured. Bitnodes only counts reachable listening nodes (not all full nodes). Pool data is Sybil-vulnerable and only covers pools with public identities.
  • 3 Usage metrics understate real network activity as more volume migrates off-chain to the Lightning Network, Fedimints, and custodial rails. On-chain velocity is a lower bound.
  • 4 Weights are editorial judgment, not derived from data. The default security-heavy allocation reflects the view that an unsecured or centralized network makes usage metrics irrelevant. Adjust them above to reflect your own view.

Score Bands

75–100 ROBUST
All major dimensions healthy relative to recent history. Network is well-secured, decentralized, and actively used.

50–75 HEALTHY
Normal operating range. Minor stress in one or two dimensions but no systemic concern.

25–50 WEAK
Several dimensions below historical norms. Sustained weakness warrants attention.

0–25 CRITICAL
Multiple dimensions at historical lows simultaneously. Rare and typically coincides with severe market stress.
